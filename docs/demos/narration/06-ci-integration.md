docgen fits into your existing development workflow with minimal setup.

The pre-push hook runs docgen validate pre-push on every git push that touches the demo directory. This catches broken recordings before they reach the remote. Stream presence, drift, and narration lint are all checked in under three seconds.

For GitHub Actions, the CI workflow runs linting with ruff, unit tests across Python three-ten through three-twelve, and Playwright end-to-end tests for the wizard GUI. The e2e tests launch a real Flask server, navigate the setup and production views, edit narration, and verify persistence.

The full pipeline, docgen generate-all, is designed for reproducibility. Given the same narration markdown and visual sources, it produces identical output. This means you can regenerate all demo assets in CI after a code change and diff the results.

For teams, the wrapper shell scripts generated by docgen init mean everyone uses the same commands. generate-all dot sh, compose dot sh, and validate dot sh all delegate to docgen. New team members install docgen, run the scripts, and everything works.

That is the full loop. Write narration in markdown. Generate audio with TTS. Render visuals with Manim and VHS. Compose with ffmpeg. Validate before you push. And the wizard helps you write the narration in the first place.
